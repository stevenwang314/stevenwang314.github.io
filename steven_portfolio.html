Song recommendations using Spotify API

A person has songs that they like to listen over and over. But what if they want to listen to new songs?

Assuming the user has a Spotify Account, the goal of this app is for the user to be able to compile a list of songs they like. The user can either use their own Spotify Account to use their playlist as their list or they can use a search engine in Spotify to grab songs of their choice. In the contrary, the user can create songs they dislike of their choice.

Once the user decides what the like or dislike, they can pick up to 5 songs or artists as a selector to feed to Spotify to generate a list of songs that will be recommended from there. In addition, having more songs on the list does affect the song recommendations even though they are not selected. The user can filter out songs based on popularity. Songs that are not popular will never be selected.

The user can repeat this step over and over by changing the 5 songs or artists of their own liking. The list of recommended songs will only add songs that are not on the list.

When the user decides he/she is ready to accept the list, they can click a button that adds ALL of the recommended songs except the songs they dislike into a new playlist directly at an instant. The user then can play the playlist without having to add the songs manually which takes a excruciating amount of time.

Implementations

Language: Javascript, HTML, and CSS Libraries: React, Bootstrap, Jquery, Spotify API, RESTful API, and Browser’s local storage

In order to use Spotify API and their RESTful principles, an access token is required. To get an access token, first I must follow the Authorization Guide by first obtaining an access token through mainly Client Credentials Flow. Second, I also have to create a Spotify app on their dashboard to obtain several sensitive information such as Client Key and Client Secret. I also need a Spotify Account to gain authentication to finally get the Access Token.

To securely hide the sensitive Client ID and Client Secret, it is stored in a hidden environment variable behind the backend through Node.js. It uses Express to accept request from our front end where it calls GET/POST methods to our back-end we have and receive the messages. There we have two API functions which we have that can authorize the users Spotify account into the app and another one which is used to obtain an access token by using Axios library. Both of these function utilizes the Client ID and Client Secret to hook into Spotify API. Once we get the access token, we finally can use it to call our API calls from Spotify.

Getting the access token is automatically called once the user is able to click refresh permission but also an additional time once the access token expires, which is in 1 hour, as there is also a refresh token to get another access token which only happens once.

In the front-end part of our app, we use the fetch() function to call our Spotify API functions with our access token obtained to be able to search for songs, artists, albums, recommending songs, getting song audio features, and then adding songs to playlist. To be able to constantly update the dynamic list on the webpage, we need to use React whenever we modify the list to update the displayed dynamic list.

The input we feed it to the Spotify Song recommendation algorithm uses the average of all songs features that are on the favorite songs list from the user followed by the 5 “seeds” from the users’ songs/artists to generate recommended songs. Finally, once the song recommendation list is made, we use the save song to playlist using the Spotify API to directly add them to the playlist directly.

Bonus:

Language: C++, Libraries: C++ Standard Library, WinHTTP

I have used C++ to interact through RESTful principals to accessing Spotify API requests. To be able to perform RESTful API calls, I used WinHTTP API to do HTTP requests. As C++ does not have an official way to parse JSON, I created my own JSON parser to be able to parse the output of JSON obtained from the results. Auto Battle Simulator

A fighting simulation project that features two teams with a given specific number of units to clash against each other with no external input from the user at all.

All units are automatically controlled by the computer AI. It controls the units’ movements that will move toward the opposing team, automatically attacking the nearest enemy as well as casting their own abilities.

There are more than 30 different types of units to choose from, each unit containing a unique ability exclusive to that particular unit.

Depending on the unit, some units are able to attack at a longer distance where as some of the units are able to hit up close.

The winner is when one team is able to eliminate all other units from the opposing team.

There are a variety of modes to choose from in which they all have specific purposes.

The program uses Microsoft’s Windows32 API to create a window program, Direct2D to render images, shapes, DirectWrite to render Text, and XAudio2 to play audio. The unit data is stored as a XML file and is ready using Microsoft XML (MSXML) API in a Document Object Model(DOM) style.

The entire project uses the Object Oriented Programming (OOP) idea to store them as classes. For the unit class, it uses the Composition methodology where we can make multiple different kinds of units without adding more classes. Inside the Unit class, it contains a Skill class. The Skill contains several Effect classes which can be interchanged between units. For instance, an effect such as firing a projectile, creating a hitbox, apply buffs to itself or debuffs to enemies or even heal a unit. The unit class also contains a Buff class that stores all buffs/debuffs which affect their performance in the fight.

All the Graphical User Interface used in the game are hand-made such as Button, Sliders, Checkbox, and Editboxes which are taken from previous projects.

Tower Defense

A tower defense game with 20 towers each having specific unique properties to handle against 18 different enemies in a map. There are enemies that can weaken towers by making them shoot slower, weaker projectiles, reduce their enemy detection range, as well as not letting them do anything at all. There are also enemies that help boost enemies speed and even force towers to attack a specific enemy. For towers, there are some towers that requires manual casting to deal powerful attack as well as some towers that boost other towers performance.

Other than that, normal tower defense rules still apply. Spend gold to build towers and do not let enemies reach the exit path. Each enemy that escapes to the end of the path costs 1 health and the game ends when all health is lost. There isn’t an end to the game but to get as far as possible.

Implementations

Language: C++, Libraries: C++ Standard Library, Windows32 , Direct2D, XAudio2, MSXML APIs

The program uses Microsoft’s Windows32 API to create a window program, Direct2D to render images, shapes, DirectWrite to render Text, and XAudio2 to play audio.

Each Turret, enemies, and projectiles uses Object Oriented Programming (OOP) and are put in classes. To differentiate different turrets and enemies, each turret and enemy use class inheritance to override the base turret and enemy class.

For towers that shoot at an angle and to collide with enemies, it uses the Separate Axis Theorem to perform angled concave shapes.

For faster collision detection when checking through a larger enemy list, I have implemented a Spatial Hash data structure which is a hash table that divides portions of the world space into a list for that particular location. To access the list at a specified location, it needs a 2D vector key containing a x and y value. By implementing the Spatial Hash, the overall game performance is faster mainly because the amount of collision checks between projectile and enemies is significantly reduced.

Sudoku

A number placement puzzle game where the goal is to fill a partially completed grid using numbers from 1 to 9 to finish the puzzle. The sole rule of this puzzle game is that each row, column, and region in a 9x9 grid is to contain all the numbers from 1 to 9.

The person can use numbers 1 to 9 to fill in the empty cells in hopes of filling the entire puzzle. Sometimes it can’t be done easily, in which the player can mark candidates from 1 to 9 in which tells the player that they can be possible numbers that fill in the cell. All advanced techniques utilized candidates which narrows down possible candidates to a few in which the user can finish the puzzle off easier. Or if the user wants to save time, the person can use auto-fill candidates to fill ALL possible candidates in the entire puzzle so they don’t have to write themselves.

In case of an error, the program will detect errors whenever the player inputs an incorrect number such as two numbers in the same row, column, or region. An undo/reverse button to either redo or revert changes made into the puzzle.

There are many sample puzzles that can be loaded from an xml file for the player to solve. If that is not enough, the system can generate a new puzzle with a unique solution of its own providing nearly infinite amount of sudoku puzzles a person can solve.

Implementations

Language: C++, Libraries: C++ Standard Library, Windows32 , Graphics Development Interface (GDI), MSXML APIs

In order to solve sudoku puzzles (including the hardest sudoku puzzles out there). or generate new sudoku puzzles, I have used the modified Backtrack Algorithm. The modifications to the Backtrack Algorithm are that first it will eliminate candidate numbers that does not fit in either row, column, or region in that particular cell. Next it will identify Naked/Hidden Singles and Intersection Row/Column techniques that will further reduce candidates possible. This significantly improves the algorithm speed as there will be less possible cases possible in a particular possible.

To be able to solve sudoku puzzle using numbers logic, it is much harder to implement then modified Backtrack algorithm. To be able to get started, I had to create several arrays which are number of candidates, list of singles, pairs, triples, and quads, as well as candidates that remain in all row, column, and region. This allowed me to make Naked/hidden single, pairs, triples, and quads, intersections (pointing/claiming), X, XY, XYZ, W wings, Swordfish, and Simple Coloring. This has been extremely challenging for me as it is extremely hard to debug and find errors due to a 2d array of 9x9 making it hard to find which index were changed and so on.

Guitar Chord Detector

The Guitar chord detector is used to detect the type of chord when the user plucks a chord from the guitar. The results appears on a Smartphone App through a Bluetooth from the HC-05 Bluetooth Module inside our hardware where it determines which type of chord has been struck.

To perform this, the person first connects the guitar into the hardware and attempt to strum a chord. Then, once the processing finished, slide the guitar pick in the smartphone app to obtain the detected chord.

Implementations

Language: C (Java for Android app), Libraries: MSP432 library

It uses a MSP432 hardware that is capable of doing a Fast Fourier Transform on hardware which signficantly increases it performance as the computation of the FFT is done in one clock cycle. The usage is to convert 4096 ADC samples into a frequency samples. In the frequency samples, we select the 10 highest peaks from the amplitude of our frequency.

We also used K-Nearest Neighbors algorithm which helps us determine how similar our FFT results are compared to other FFT results. This helps us match up with given frequency of known chords easily.

Seizure Flash Blocker

Made in Hackathon of fall 2018 in 48 hours, the goal of the project is to replace quick flashing strobe of lights in certain videos into a static color to protect the viewer from collapsing from a seizure. In this program we used Google Chrome and a youtube video (as demonstration) from a clip from Pokemon (Dennō Senshi Porygon) that caused hundreds of people to be sent to hospitals due to seizures caused in that episode.

A seizure (or a photosensitive epilepsy) is made from constantly flashing colors or regularly repeating patterns

Implementations

Language: C/C++, Libraries: C++ Standard Library, Windows32 , Graphics Development Interface (GDI)

The program is ran in background, there is no window that shows up

It uses an algorithm which determines the aggregate color difference which compares the difference of color between the current frame and the last frame. The color has three channels of Red, Green, and Blue dimension which is then pointed in space. We can compute the distance between two colors through the distance formula. If the distance is bigger, it is shown that the color is changed extremely. This is for one pixel. To be able to cover the entire screen, we have to sum the color distances over the entire region to compute an aggregate between two frame.

It records number of times within 200ms period that color difference reaches a specific threshold. If this threshold activates 3 times, it is marked as flashing. We used GDI (Graphics Development Interface) to draw a red rectangle over the region

System on Chip Design - Tramel Blaze

Tramel Blaze is a modified version of PicoBlaze architecture where it is able to execute 16bits of instructions compared to PicoBlazes 8 bit instructions. It contains components of 16 bit registers, SRAM (Static Random Access Memory) with 32,768 address, TSI (Technology Specific Information), and UART (Universal Asynchronous Receiver Transmitter) modules that when used, it can store/load data from memory as well as receiving/transferring data from another device. Our Verilog code is then programmed onto a Nexys4 Board. There, it is fed lines of assembly language instructions and its output appears on the 8 digit LED. The buttons go through each register and also traversing through instructions

Implementations

Language: Verilog / Assembly Language

AISO – The Asynchronous Input Synchronous Output is a module that turns an Asynchronous signal into a Synchronous Output. The main use of the module is to prevent an unstable signal know as metastability.

TSI – The Technology Specific Instantiation is a module that makes sure that the I/O of the chip meets electrical and timing requirements of the external interface.

PED – The positive edge detect simply detects whether a bit changes from low (0) to high(1)

Address Decoder – The address decoder outputs control units based on the given input varying from address and read/write strobe.

UART - The Universal Asynchronous Receive Transmitter has two component: The transmit side and the receive side.

The transmit engine uses a Baud Decoder to translate 4 bits of input to a given selected Baud Rate. The larger the baud rate, the less clock cycles it takes to reach the Bit time counter. See the table in Chip Specification for more details.

It uses the Shift Register to shift 1 bit to the right. The Least Significant bit is then sent to the TX signal. The contents contain a start bit and stop bit which is a fixed set of bits that tells the UART to start/stop processing the bits.

The receive engine is much more complex than transmit engine. It is not a voluntary process and will wait until the transmit data from the other peripheral starts sending data to the receive side of this peripheral.

Not all data will be transferred successfully from one side to another. We have to flag Parity, Framing, and Overflow errors as a way to check whether we are able to successfully transfer all of its bits from the other peripheral transmitter data to our receiver data.

Single Cycle Mips Processor

Language: Verilog

A Single Cycle Mips Processor is used to execute assembly instructions in one single clock cycle. There are many different instructions that affect registers and values in the memory address in the processor. We extended our Single Cycle Mips Processor by adding more instructions into the processor such as Load Word Increment (LWINC), Exchange (Xch), Multiple (MULT), Divide(DIV), Jump Bit (JB), Jump Carry( JC), and etc. By adding more instructions, we have to modify the structure of our processor in order to incorporate the new instructions to ensure that these extension instructions can be resolved with a single clock cycle.
