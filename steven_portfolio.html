Sudoku

A numbers puzzle game that the goal is to fill the entire puzzles with numbers from 1 to 9 to finish the puzzle. Each number can only appear once from 1-9 at row, column, and region for each cell.

  • It uses the Backtrack Algorithm (optimized) to solve any Sudoku puzzle. It will automatically first handle Naked/Hidden Singles, and Intersection Row/Column first significantly improving the algorithm speed.
  • New Puzzles can be generated using Backtrack Algorithm with a variety of difficulty.
  • Built in hint that uses human technique logics that can solve all but the hardest Sudoku Problems out there
  • Allows the player to input cell numbers or candidate numbers
  • Pre-made puzzles that can be loaded from a xml file containing multiple sudoku puzzles, use the arrow keys to select a different type of puzzles.

Auto Battle Simulator v2

The enhanced and refactored version of the previous. It has many type of modes that the units can go through.

  • There are 50 different units to choose from with different abilities
  • Each unit can evolve up to 7 times to gain more strength
  • Each unit can equip gear to also get stronger.
  • Different sets of modes
    • Arena - A group of 10 heroes attempt to fight another group of 10 heroes.
    • Boss Fight - Fights a boss. The main goal is to deal as much damage as possible. But the more damage it takes, the stronger it becomes
    • Dungeon - Battle against 10 floors worth of enemies and reach to the very end.
    • Adventure - A endless battle that helps units get stronger by defeating endless amount of enemies.
    • The player can also get experience and gold by not leaving the program on.

Auto Battle Simulator

A Simulation that features two sides fighting against each other except no user input is involved. All units are automatically controlled by the game resulting in relax-free gameplay.

  • By default, units will automatically head to the opposing side walking normally.
  • Units engage with enemy units when it reaches a specific distance, units that can attack from afar will engage further than units that can hit up close.
  • There are 20 different type of units, each unit contains specific ability exclusive to that particular unit.

Tower Defense

A tower defense game made from scratch using C++ programming language. It uses windows32 Operating System API to handle opening a window and Direct2D API to render graphics onto the window.

  • Just like other Tower Defense games, the goal is to not let enemies reach to the end of the path
  • There is no end to the game, the game ends when all Health has been depleted. Each enemy that reaches the end of the path which is on the other side reduces Health by 1.
  • Players build turrets by using gold, which is obtained by defeating enemies. Gold can also be used to upgrade towers by up to 10 times.
  • 20 different kinds of towers to use
  • 18 different enemies to encounter and to beat

2d Platformer RPG

A 2D platformer with RPG elements. It uses windows32 Operating System API to handle opening a window and Direct2D API to render graphics onto the window.

  • A built-in Inventory support. The player can equip items and use potions
  • A shop that allows the player to buy and sell items.
  • Many different areas to explore with a variety of enemies. And even 1 big enemy.
  • 4 different skills the player can use to tackle enemies.
  • Breakable boxes that the player can destroy in the beginning
  • The game is saved when the game closes. The file will be saved in a XML file. It uses MSXML API from microsoft to write and read XML files.

Grid Adventurer

First program made with Javascript using <canvas> element in HTML as a Arcade game

  • The goal of the game is to get as far as possible while collecting loads of loot along the way
  • The player can move with arrow keys and can interact with other objects with arrow keys
  • Each action consumes energy. Using all energy will result in fatigue which will rapidly consume health
  • The main goal of each stage is to find the flag, which proceeds to the next stage. To help out with that, there are arrow signs that point to (but not exactly) the red flag to make it easier
  • The player can collect gold and chests to gain more points during the stage.
  • As the player reaches higher stages, there will be more obstacles, hazards, and less likely to find any arrows and supporting items. However, the player gains more points at higher stages as a result.
  • The game is over when the player loses all of their health
  • The demo of this game can be accessed here or on the Demo Game link!

Accessing Spotify API

  • Used Various request types such as GET, POST, REQUEST, ADD, DELETE.
  • Created access token each time whenever it is needed to access API calls
  • Created a JSON Parser that parsed returned JSON objects into objects using std::map and std::variant (any data types)

Tramel Blaze

Tramel Blaze is a modified version of PicoBlaze architecture where it is able to execute 16bits of code instructors. It contains a register, memory, RAM, and UART modules that help store data, receive/transfer data to another device. This is made using Xilinx ISE with Verilog as the language. It is then programmed onto a Nexys4 Board. It is done by feeding it lines of instructions and its output appears on the 8 digit LED. The buttons go through each register and also traversing through instructions
