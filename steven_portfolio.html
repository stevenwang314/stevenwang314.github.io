Grid Adventurer

Language: Javascript, HTML, CSS

My first program made using Javascript with the <canvas> element. The player controls a character that adventures throughout a 25x25 grid in hopes to finding a red flag. However, it is much easier said than done. To get there, the player encounter enemies, obstacles, and dangerous hazards, though there are usable items and loot comes along the way that assist the player.

Moving, destroying objects, and attacking enemies may be simple but they are limited. Each of those actions will deplete the players resources in which the player has finite amount of actions before the player reaches their own demise.

This program uses the Document Object Model (DOM) API to interact with elements in html. In this case, we want to access the <canvas> element from Javascript. Each stage contains completely random placements of enemies, objects, and obstacles in which no two stages are exactly the same.

Song recommendations using Spotify API

Language: Javascript, HTML, and CSS Libraries: React, Bootstrap, Jquery, and Browser’s local storage

Sometimes a person would repeat listening to the same songs mainly that the songs they like are far much better than songs that are out of their comfort zone. To partially alleviate this issue, a song recommendation based on the last recent songs they played is needed.

In this program, the goal is to provide a recommended song list based on the list of artists or songs that the person likes. The demo of this game can be accessed here or on the Spotfiy Demo above! and Github Source

  • Made with Javascript, HTML, and CSS followed by helpful libraries such as React, Bootstrap, Jquery, and browser’s local storage.
  • Interacted with official Spotify API through their RESTful API by first obtaining a access token through Client Credentials or Authorization Code.
  • It does not require an account to use this, though an account is helpful since the generated playlist can be added to the player’s new playlist.
  • The person can choose up to 5 tracks/artists combined and the spotify recommendation will run behind the scenes to provide up to 100 songs that are related to the person liking. This can be done multiple times as each unique song adds to a bucket.
  • Once the person is satisfied after finding many songs in their recommendation list, the person can save it to a brand new playlist to listen to all the recommendation songs at a instant. No adding one by one, the program does it all at a instant!
  • The person can add as many songs as they want in their list as each song does somewhat affect the recommendations.

Auto Battle Simulator

Language: C++, Libraries: C++ Standard Library, Windows32 , Direct2D, XAudio2, MSXML APIs

A fighting simulation project that features two sides fighting against each other except that no user input is involved. All units are automatically controlled by the game resulting in hands-free gameplay. It does get tiring when the player has to spend countless hours on trying to achieve a goal in a game, but why do that when the game can be played all on its own while the player can sit back and do something else. The battle ends when either side has no units remaining.

There are more than 20 different types of units to choose from, each unit contains an specific ability exclusive to that particular unit. The abilities are also automatically activated.

A computer artificial intelligence controls all units. By default automatically head to the opposing side walking normally.

Units engage with enemy units when it reaches a specific distance, units that can attack from afar will engage further than units that can hit up close.

There is a first iteration of this project where the units are shapes with different abilities. This demonstrates a never ending battle where new units constantly spawn to replace any that were slained. The current iteration of this projects contains differet kinds of modes to play so it does not look monotone style of gameplay.

Tower Defense

Language: C++, Libraries: C++ Standard Library, Windows32 , Direct2D, XAudio2, MSXML APIs

A tower defense game with 20 towers each having specific unique properties to handle against 18 different enemies in a map. There are enemies can weaken towers by making them shoot slower, weaker projectiles, as well as not letting them do anything at all. There are also enemies that help boost enemies speed and even force towers to attack a specific enemy. Other than that, normal tower defense rules still apply. Spend gold to build towers and do not let enemies reach the exit path. Each enemy that escapes takes off 1 health and the game ends when all health is lost. There isn’t an end to the game but to get as far as possible.

Sudoku

Language: C++, Libraries: C++ Standard Library, Windows32 , Graphics Development Interface (GDI), MSXML APIs

A number placement puzzle game where the goal is to fill a partially completed grid using numbers from 1 to 9 to finish the puzzle. The sole rule of this puzzle game is that each row, column, and region in a 9x9 grid is to contain all the numbers from 1 to 9.

There are sample puzzles a user can solve as well as creating brand new sudoku puzzles from scratch by simply pressing generate puzzle.

  • It uses the modified Backtrack Algorithm to solve any Sudoku puzzle (including the hardest sudoku puzzles out there). The modifications to the Backtrack Algorithm are that first it will eliminate candidate numbers that does not fit in either row, column, or region in that particular cell. Next it will identify Naked/Hidden Singles and Intersection Row/Column techniques that will further reduce candidates possible. This significantly improves the algorithm speed as there will be less possible cases possible in a particular possible.
  • The modified Backtrack Algorithm is also used to generate a brand new sudoku puzzle that always results in one unique solution.
  • A built-in hint that uses several human technique logics that can solve all but the hardest Sudoku Problems out there.
  • Detects errors whenever the player inputs a incorrect number.
  • Pre-made puzzles that can be loaded from a xml file containing multiple demo sudoku puzzles, the user can use arrow keys to switch between puzzles.

Guitar Chord Detector

Language: C (Java for Android app), Libraries: MSP432 library

The Guitar chord detector is used to detect the type of chord when the user plucks a chord from the guitar. The results appears on a Android Application through Bluetooth where it determines which type of chord has been struck.

It uses a MSP432 hardware that is capable of doing a Fast Fourier Transform on hardware which signficantly increases it performance as the computation of the FFT is done in one clock cycle. The usage is to convert 4096 ADC samples into a frequency samples. In the frequency samples, we select the 10 highest peaks from the amplitude of our frequency.

We also used K-Nearest Neighbors algorithm which helps us determine how similar our FFT results are compared to other FFT results. This helps us match up with given frequency of known chords easily.

Single Cycle Mips Processor

Language: Verilog

A Single Cycle Mips Processor is used to execute assembly instructions in one single clock cycle. There are many different instructions that affect registers and values in the memory address in the processor. We extended our Single Cycle Mips Processor by adding more instructions into the processor such as Load Word Increment (LWINC), Exchange (Xch), Multiple (MULT), Divide(DIV), Jump Bit (JB), Jump Carry( JC), and etc. By adding more instructions, we have to modify the structure of our processor in order to incorporate the new instructions to ensure that these extension instructions can be resolved with a single clock cycle.
